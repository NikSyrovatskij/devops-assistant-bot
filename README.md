# VPS Assistant Bot 🤖

**VPS Assistant Bot** — это простой и надежный инструмент для мониторинга ваших серверов через Telegram. Бот позволяет следить за состоянием нескольких VPS в реальном времени, используя SSH-подключение. 

Главная особенность — **безагентный мониторинг**. Вам не нужно устанавливать дополнительное ПО на целевые серверы.

## 🚀 Основные возможности

- 📊 **Мониторинг в один клик:** Данные о CPU, RAM, Disk и Uptime со всех серверов сразу.
- ➕ **Удобное управление:** Добавление и удаление серверов через диалог в боте.
- ⌨️ **Интерактивное меню:** Удобные кнопки вместо ручного ввода команд.
- 🔒 **Безопасность:** Доступ к боту ограничен по вашему Telegram ID.
- 🐳 **Docker:** Быстрый запуск всей системы в контейнерах.

## 📋 Какие данные выводит бот?

Для каждого сервера вы получаете:
- **Статус:** Online/Offline.
- **Uptime:** Время работы без перезагрузки.
- **CPU:** Модель процессора и средняя нагрузка (Load Average).
- **RAM:** Использование оперативной памяти в MB и %.
- **Disk:** Свободное место на корневом разделе.

## 📦 Быстрое развертывание

### 1. Клонирование репозитория
```bash
git clone https://github.com/NikSyrovatskij/vps-assistant-bot.git
cd vps-assistant-bot
```

### 2. Настройка переменных окружения
Создайте файл `.env` на основе примера:
```bash
cp .env.example .env
```
Откройте `.env` через `nano .env` и заполните:
- `BOT_TOKEN`: Токен от @BotFather.
- `ADMIN_ID`: Ваш числовой ID в Telegram.

### 3. Запуск
```bash
docker-compose up -d --build
```

## 🕹 Команды бота
- `📊 Статус серверов` — Опросить все серверы в базе.
- `➕ Добавить сервер` — Запустить пошаговый диалог добавления.
- `📋 Список серверов` — Показать все сохраненные узлы и их ID.
- `/remove_server ID` — Удалить сервер из базы по его идентификатору.
